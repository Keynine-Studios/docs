Organization settings in Key Nine Studios define how your workspace presents itself and which guardrails apply by default across all projects. Changes here often require admin rights and may notify all members—communicate important changes before making them.

## Update your org profile

Update your organization's display name and public-facing details shown in invites and notification footers. Align the spelling with your legal entity name where DPA references and invoices matter.

<Note>
Use brand-approved logos and run color contrast checks for accessibility when adding visual assets. Store source files in your brand portal so future admins do not have to hunt for attachments.
</Note>

## Set workspace defaults

Configure default project visibility and automation limits for new projects created by members. Defaults reduce setup overhead for large teams rolling out multiple projects at once.

<Tip>
Pair workspace defaults with project templates so teams inherit sensible labels, sections, and automation caps without starting from zero every time.
</Tip>

## Enforce policies

Admins can set the following workspace-level policies:

| Policy | Description |
|---|---|
| Require MFA | Forces all members to enable multi-factor authentication |
| Restrict invite domains | Limits invites to approved email domains |
| Data residency | Sets which region stores workspace data (where available) |

### Document exceptions

Create a clear process for requesting exceptions—contractors, acquisitions, or partner accounts—so your security team can approve without inventing a new workflow each time.
