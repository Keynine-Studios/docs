OAuth apps in Key Nine Studios allow you to build integrations that act on behalf of users without sharing their passwords. Treat client secrets like production credentials—rotate them on a schedule and restrict who can create new apps in your workspace.

## Register an OAuth app

<Steps>
  <Step title="Open Developer settings">
    Navigate to **Settings → Developer → OAuth Apps** and click **New app**.
  </Step>
  <Step title="Name and describe your app">
    Use a name users will recognize in consent screens. Add a short description explaining what the app does.
  </Step>
  <Step title="Note your credentials">
    Copy the client ID and client secret immediately. The secret is shown only once—store it in your secrets manager before closing the page.
  </Step>
  <Step title="Set redirect URLs">
    Add redirect URLs for each environment. Use HTTPS endpoints only. Register separate entries for staging and production.
  </Step>
</Steps>

<Warning>
Maintain separate OAuth apps for staging and production environments so tokens never cross accidentally.
</Warning>

## Configure scopes

Request the smallest scope set that satisfies your feature requirements. Unnecessary scopes alarm security reviewers and violate the principle of least privilege.

<Tip>
Write user-facing copy that explains why each scope is needed. Both enterprise security reviewers and end users during consent will ask.
</Tip>

## Use PKCE for public clients

For mobile apps and single-page applications where you cannot safely store a client secret, use the PKCE extension. PKCE reduces interception risk by generating a code challenge at the start of the flow.
