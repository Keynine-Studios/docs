API keys in Key Nine Studios authenticate machine-to-machine calls between your servers and the platform. They are simpler to implement than OAuth but carry a higher leak risk—never embed them in browser-side code or mobile apps.

## Create an API key

Navigate to **Settings → Developer → API Keys** and click **New key**. Give the key a descriptive label so you can identify which service uses it during rotation.

<Warning>
Store API keys in a secrets manager (such as AWS Secrets Manager, HashiCorp Vault, or your CI/CD secrets store) immediately after creation. Never commit keys to source control or paste them into chat.
</Warning>

### Scope your keys

If Key Nine Studios supports scoped keys on your plan, split read vs write access. A compromised read-only key cannot mutate data, which limits the blast radius of a leak.

## Rotate keys safely

Use dual-key rotation to avoid hard cutovers:

<Steps>
  <Step title="Issue a new key">
    Create a new API key with the same scope as the key you are replacing.
  </Step>
  <Step title="Deploy the new key">
    Update every service, cron job, and CI pipeline that uses the old key to use the new one.
  </Step>
  <Step title="Verify traffic">
    Confirm all services are authenticating successfully with the new key before revoking the old one.
  </Step>
  <Step title="Revoke the old key">
    Revoke the old key from **Settings → Developer → API Keys**. Revocation is immediate.
  </Step>
</Steps>

<Tip>
Document which cron jobs and CI pipelines use which API key. A runbook prevents rotation from missing a dusty integration.
</Tip>

## Revoke a key

Revoked keys stop working immediately. Before revoking, update all dependent services to avoid downtime.
