Search is the fastest path from "where was that?" to the right page—especially as your Key Nine Studios workspace grows. Combine query syntax, filters, and recent items to cut noise and land on the canonical answers your team has already written.

## Global search

Open global search with **⌘P** (Mac) or **Ctrl+P** (Windows/Linux). Search spans titles, page bodies, and selected metadata. Filters narrow results by owner, modification date, and content type.

### Search operators

| Operator | Effect | Example |
|---|---|---|
| Quoted phrases | Exact match | `"approval workflow"` |
| Minus prefix | Exclude term | `onboarding -deprecated` |
| Filters | Narrow by attribute | `author:alice type:page` |

### Result ranking

Results favor recent edits and stronger title matches. If something ranks unexpectedly low, try a more specific phrase or open the containing project and browse by folder structure.

## Recent items

Recent items appear in the sidebar for fast return visits. Clear your recents before demoing or handing off a laptop to avoid leaking internal project names on a shared screen.

### Pin favorites

If your client supports pinned items or favorites, use them for runbooks and onboarding hubs you open daily—faster than retyping the same query every time.

## Search tips

- Use quoted phrases for exact matches on technical terms.
- Use minus terms to exclude boilerplate pages that appear in every result.
- When you find a canonical page, add a short internal link from related articles so the next person finds it in one click instead of searching repeatedly.

<Tip>
If content is missing from search entirely, file a docs task or create a stub page. Search cannot surface what does not exist yet.
</Tip>
