Keyboard shortcuts in Key Nine Studios reduce context switching by keeping your hands on the keyboard for formatting, navigation, and collaboration actions. The tables below cover common defaults—your organization may publish a cheat sheet for new member onboarding.

<Note>
On Windows and Linux, **Ctrl** replaces **⌘** wherever both are listed below.
</Note>

## General shortcuts

| Action | Mac | Windows / Linux |
|---|---|---|
| Command palette | ⌘K | Ctrl+K |
| Global search | ⌘P | Ctrl+P |

If a shortcut conflicts with your browser or OS, disable the conflicting browser extension or remap the shortcut inside **Settings → Accessibility** if your client supports it.

## Editor shortcuts

| Action | Mac | Windows / Linux |
|---|---|---|
| Bold | ⌘B | Ctrl+B |
| Add comment | ⌘⌥M | Ctrl+Alt+M |

### Selection and block behavior

Many editors support **⌘D** / **Ctrl+D** for multi-cursor or duplicate-line behavior. Check in-app shortcut hints because behavior can vary between block types (paragraph vs code block vs table).

## Accessibility notes

If reduced-motion or sticky keys are enabled at the OS level, chord shortcuts may require slower presses. Screen reader users should verify that focus lands in the expected region after palette commands.

### Input method editors (IME)

IME users composing CJK text may need to disable certain modifier-key chords temporarily or use the command palette instead of direct shortcuts that intercept modifier keys.

## Customize shortcuts

Remap shortcuts in **Settings → Accessibility** on supported clients. Export your mapping before switching machines so you can recreate your configuration on a new device.
